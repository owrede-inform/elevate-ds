# ColorRamp Component

A powerful documentation site component for dynamically displaying ELEVATE design system color tokens using flexible pattern matching and customizable templates.

:::info Documentation Site Component
This is a custom component built specifically for this documentation site. It is **not** part of the ELEVATE Core UI library.
:::

## Overview

The ColorRamp component automatically discovers and displays color tokens from the ELEVATE design system based on selector patterns. Perfect for creating dynamic color documentation, style guides, and design token exploration tools.

**Key Features:**
- üéØ **Smart Pattern Matching** - Use wildcards and flexible selectors
- üîÑ **Multiple Sorting Methods** - Semantic, brightness, alphabetic, and hue sorting
- üé® **Custom Templates** - Create your own display layouts
- üì± **Responsive Design** - Works on all screen sizes
- üöÄ **Dynamic Discovery** - Automatically finds tokens in CSS

### Selector Patterns

The `selector` prop supports flexible pattern matching:

| Pattern | Matches | Example |
|---------|---------|---------|
| `primitives-color-blue-*` | All blue primitives | blue-50, blue-100, blue-500, etc. |
| `primitives-color-*-500` | All 500-weight colors | blue-500, red-500, green-500, etc. |
| `*-color-*` | All color tokens | Any color from any category |
| `component-button-*` | Button-specific colors | button-fill, button-text, etc. |

| Variable | Description | Example |
|----------|-------------|---------|

## Usage Guidelines

### Best Practices

1. **Start Broad**: Use `*-color-*` to explore available tokens
2. **Filter Progressive**: Add specific patterns to narrow results
3. **Choose Appropriate Sorting**: Use semantic for documentation, brightness for accessibility
4. **Group for Clarity**: Group by color family when showing multiple colors
5. **Custom Templates**: Create custom layouts for specific use cases

### Common Patterns

```tsx
// Explore all available colors
<ColorRamp selector="*-color-*" />

// Document a specific color family
<ColorRamp selector="primitives-color-blue-*" sortBy="semantic" />

// Show base colors only
<ColorRamp selector="primitives-color-*-500" orientation="horizontal" />

// Accessibility analysis
<ColorRamp selector="primitives-color-gray-*" sortBy="brightness" />

// Component-specific colors
<ColorRamp selector="component-button-*" showDescription={true} />
```

## Implementation Details

The ColorRamp component:

1. **Scans CSS Properties**: Dynamically reads all `--elvt-*` custom properties
2. **Pattern Matching**: Uses regex to match selector patterns against property names  
3. **Color Detection**: Identifies color values (rgb, hsl, hex) and filters non-colors
4. **Smart Sorting**: Applies semantic understanding of design token naming conventions
5. **Template Engine**: Processes custom templates with variable substitution
6. **Responsive Layout**: Adapts to container width with CSS Grid

Perfect for design system documentation, component showcases, and interactive color exploration!

### Adding New Templates

To add a new predefined template:

1. **Create Template HTML**: Add your template to `/src/components/ColorRamp/templates/new-template.html`
2. **Add to Registry**: Update the `templateRegistry` in `index.tsx`:
   ```typescript
   const templateRegistry: Record<string, string> = {
     // ... existing templates
     'new-template': `<div>Your HTML template with {{variables}}</div>`,
   };
   ```
3. **Update TypeScript**: Add the template name to the `ColorRampProps` interface:
   ```typescript
   template?: 'default' | 'compact' | 'detailed' | 'custom' | 'simple' | 'badge' | 'card' | 'table-row' | 'accessibility' | 'swatch-grid' | 'new-template';
   ```
4. **Document**: Add examples to this documentation page

### Template Guidelines

- **Use Variables**: Include relevant `{{variable}}` placeholders
- **CSS Classes**: Leverage pre-styled classes like `.color-swatch`, `.token-info`
- **Responsive**: Ensure templates work on mobile devices
- **Semantic HTML**: Use proper HTML structure for accessibility
- **Test**: Verify with different color sets and grouping options
