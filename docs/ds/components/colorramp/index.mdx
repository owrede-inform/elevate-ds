# ColorRamp

Automatically displays ELEVATE design system color tokens with pattern matching and customizable templates.

:::info Documentation Site Component
This is a custom component built specifically for this documentation site. It is **not** part of the ELEVATE Core UI library.
:::

<TocLinks />

## Overview

The ColorRamp component discovers and displays color tokens from CSS using selector patterns. Ideal for color documentation and design token exploration.

**Key Features:**
- **Pattern Matching** - Wildcards and flexible selectors
- **Multiple Templates** - Default, compact, table, card layouts
- **Sorting Options** - Semantic, brightness, alphabetic sorting
- **Responsive Design** - Works on all screen sizes
- **Dynamic Discovery** - Automatically finds tokens in CSS

## API Reference

### Props

| Prop | Type | Default | Required | Description |
|------|------|---------|----------|-------------|
| `selector` | `string` | - | ✅ | Pattern to match color tokens (supports wildcards) |
| `exclude` | `string` | - | ❌ | Pattern to exclude from results (supports wildcards) |
| `title` | `string` | - | ❌ | Display title for the color ramp |
| `template` | `string` | `'default'` | ❌ | Display template (default, compact, table, card) |
| `sortBy` | `string` | `'semantic'` | ❌ | Sorting method (semantic, brightness, alphabetic) |
| `includeShades` | `number[]` | - | ❌ | Only show specific shade numbers |
| `showHexValue` | `boolean` | `false` | ❌ | Show hex/rgb color values |

### Selector Patterns

| Pattern | Matches |
|---------|---------|
| `primitives-color-blue-*` | All blue shades (50, 100, 200...900) |
| `primitives-color-*-500` | All base colors (blue-500, red-500, etc.) |
| `*-color-*` | All color tokens |

## Usage Guidelines

**Common Patterns:**
- `primitives-color-blue-*` - Document a specific color family
- `primitives-color-*-500` - Show base colors across families
- `*-color-*` - Explore all available colors
- `exclude="primitives-color-gray-0"` - Remove specific tokens from results

**Best Practices:**
- Use `template="compact"` for inline documentation
- Use `includeShades={[300, 500, 700]}` to focus on key shades
- Add `showHexValue={true}` for developer reference
- Use `exclude` to filter out unwanted tokens (e.g., gray-0, transparent)
- Use descriptive titles to provide context
