# Radio

Radio buttons enable users to select exactly one option from a group of mutually exclusive choices. The ELEVATE Radio component ensures accessible, consistent single-selection interfaces that integrate seamlessly with forms and the design system.

## Overview

### When to Use Radio Buttons
- **Mutually exclusive choices**: When users must select exactly one option from a group
- **Preference settings**: Setting default options or user preferences
- **Configuration options**: Choosing between different configuration modes
- **Survey responses**: Single-answer survey questions and forms
- **Payment methods**: Selecting one payment option from available choices

### When NOT to Use Radio Buttons
- **Multiple selections**: Use checkboxes when users can select multiple options
- **Binary choices**: Use switches or single checkboxes for simple on/off choices
- **Many options**: Consider dropdowns or other components for large option sets
- **Optional selection**: When no selection is a valid choice, consider different patterns

## Design Principles

### Clarity Principle
**Radio button options should be clearly labeled and mutually exclusive.** Ambiguous or overlapping options confuse users about which choice to make.

### Grouping Principle
**Related radio buttons should be visually grouped and properly labeled.** Clear grouping helps users understand the relationship between options.

### Selection Principle
**Exactly one option should be selectable within each radio group.** Breaking this fundamental rule creates unpredictable interface behavior.

### Accessibility Principle
**Radio groups must be keyboard accessible with proper focus management.** Screen readers need proper group labeling and option announcements.

## Best Practices

### Organizing Radio Options

**Why this matters:** Logical organization helps users find their preferred choice quickly. Poor organization increases cognitive load and selection errors.

:::tip Do: Organize options in logical order
Present options in a meaningful sequence that helps users find their choice quickly.

```html
<elvt-radio-group label="Priority level:" name="priority">
  <elvt-radio value="high">High Priority</elvt-radio>
  <elvt-radio value="medium">Medium Priority</elvt-radio>
  <elvt-radio value="low">Low Priority</elvt-radio>
</elvt-radio-group>
```
:::

:::danger Don't: Use random or confusing organization
Avoid arbitrary ordering that makes options hard to find or compare.

```html
<!-- Avoid this -->
<elvt-radio-group label="Size:" name="size">
  <elvt-radio value="medium">Medium</elvt-radio>
  <elvt-radio value="extra-large">Extra Large</elvt-radio>
  <elvt-radio value="small">Small</elvt-radio>
  <elvt-radio value="large">Large</elvt-radio>
</elvt-radio-group>
```
:::

### Writing Clear Option Labels

**Why this matters:** Clear, distinct labels prevent user confusion and selection mistakes. Ambiguous labels lead to form abandonment and errors.

:::tip Do: Use distinct, clear labels for each option
Make labels specific and mutually exclusive so users understand their choices.

```html
<elvt-radio-group label="Delivery method:" name="delivery">
  <elvt-radio value="standard">Standard Delivery (5-7 days)</elvt-radio>
  <elvt-radio value="express">Express Delivery (2-3 days)</elvt-radio>
  <elvt-radio value="overnight">Overnight Delivery (next day)</elvt-radio>
</elvt-radio-group>
```
:::

:::danger Don't: Use vague or overlapping labels
Avoid generic labels that don't clearly differentiate the options.

```html
<!-- Avoid this -->
<elvt-radio-group label="Shipping:" name="shipping">
  <elvt-radio value="option1">Option 1</elvt-radio>
  <elvt-radio value="option2">Option 2</elvt-radio>
  <elvt-radio value="option3">Option 3</elvt-radio>
</elvt-radio-group>
```
:::

## Implementation

### Getting Started

```bash
npm install @inform-elevate/elevate-core-ui
```

### Basic Usage

```html
<!-- Basic radio group -->
<elvt-radio-group label="Contact preference:" name="contact">
  <elvt-radio value="email" checked>Email</elvt-radio>
  <elvt-radio value="phone">Phone</elvt-radio>
  <elvt-radio value="sms">SMS</elvt-radio>
</elvt-radio-group>

<!-- Radio with descriptions -->
<elvt-radio-group label="Plan selection:" name="plan">
  <elvt-radio value="basic">
    Basic Plan
    <span slot="description">$9/month - Essential features</span>
  </elvt-radio>
  <elvt-radio value="pro">
    Pro Plan
    <span slot="description">$19/month - Advanced features</span>
  </elvt-radio>
</elvt-radio-group>

<!-- Inline radio group -->
<elvt-radio-group label="Size:" name="size" orientation="horizontal">
  <elvt-radio value="small">Small</elvt-radio>
  <elvt-radio value="medium" checked>Medium</elvt-radio>
  <elvt-radio value="large">Large</elvt-radio>
</elvt-radio-group>

<!-- Radio with validation -->
<elvt-radio-group label="Required selection:" name="required" required error="Please make a selection">
  <elvt-radio value="option1">Option 1</elvt-radio>
  <elvt-radio value="option2">Option 2</elvt-radio>
</elvt-radio-group>
```

### Key Properties

#### Radio Group Properties

| Property | Type | Description |
|----------|------|-------------|
| `label` | String | Group label for accessibility |
| `name` | String | Form field name for all radios in group |
| `orientation` | `vertical` (default), `horizontal` | Layout orientation |
| `size` | `small`, `medium` (default), `large` | Size variant |
| `required` | Boolean | Marks group as required for validation |
| `error` | String | Error message to display |
| `success` | String | Success message to display |
| `disabled` | Boolean | Disables entire radio group |

#### Radio Button Properties

| Property | Type | Description |
|----------|------|-------------|
| `value` | String | Value submitted when radio is selected |
| `checked` | Boolean | Sets radio as selected by default |
| `disabled` | Boolean | Disables individual radio button |
| `name` | String | Form field name (usually set by radio-group) |

*For complete API reference including all properties, events, and methods, see the [Storybook documentation](https://elevate-core-ui.inform-cloud.io/?path=/docs/components-radio--docs).*

## Accessibility

### Essential Accessibility Features

**Proper grouping and labeling:**
- Radio groups use fieldset/legend or appropriate ARIA labeling
- Each radio button has a clear, descriptive label
- Group labels clearly describe the choice being made
- Related options are visually and semantically grouped

**Keyboard accessibility:**
- Arrow keys navigate between radio buttons within a group
- Tab moves between radio groups and other form elements
- Space key selects the focused radio button
- Selected radio buttons are clearly indicated to keyboard users

**Screen reader support:**
- Radio group labels are properly announced
- Individual radio labels are read when focused
- Selection state changes are announced
- Required status and validation messages are communicated

**Clear visual design:**
- Selected and unselected states are visually distinct
- Focus indicators meet accessibility contrast requirements
- Disabled states are clearly communicated visually
- Radio buttons maintain usable size for motor accessibility
