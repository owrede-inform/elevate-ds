# Badge

Badges are small visual indicators that help users quickly understand status, counts, and categories. The ELEVATE Badge component helps designers and developers create consistent, accessible interfaces that communicate important information without overwhelming the user.

## Overview

### When to Use Badges
- **Status indication**: Online/offline, active/inactive, approval states
- **Notifications**: Unread counts, new items, urgent alerts
- **Categorization**: Tags, labels, content classification
- **Metadata**: Version numbers, types, feature flags
- **Highlighting**: Important or featured content that needs attention

### When NOT to Use Badges
- **Primary actions**: Use Button component for main interactions
- **Navigation**: Use Link or navigation components for page transitions
- **Large text content**: Use appropriate text components for substantial information
- **Complex information**: Use Cards or other layout components for detailed data

## Design Principles

### Clarity Principle
**Badge content should be immediately understandable.** Users should be able to quickly interpret badge meaning without requiring additional context or explanation.

### Hierarchy Principle
**Badges should support, not compete with, primary content.** Badge styling should draw appropriate attention without overwhelming the main interface elements.

### Consistency Principle
**Similar information should use similar badge treatments.** Status badges, count badges, and category badges should maintain consistent styling patterns across the application.

### Accessibility Principle
**All users must be able to perceive and understand badge information.** This includes proper color contrast, meaningful text, and support for assistive technologies.

## Best Practices

### Using Semantic Colors

**Why this matters:** Color carries meaning and helps users quickly understand information. Consistent semantic color usage creates predictable user experiences.

:::tip Do: Use semantic colors that match user expectations
Apply colors that align with common conventions and user mental models.

```html
<elvt-badge variant="success">Approved</elvt-badge>
<elvt-badge variant="warning">Under Review</elvt-badge>
<elvt-badge variant="danger">Rejected</elvt-badge>
<elvt-badge variant="info">Draft</elvt-badge>
```
:::

:::danger Don't: Use arbitrary colors without meaning
Avoid color choices that confuse users or contradict established patterns.

```html
<!-- Avoid this -->
<elvt-badge variant="success">Error</elvt-badge>
<elvt-badge variant="danger">Success</elvt-badge>
```
:::

## Implementation

### Getting Started

```bash
npm install @inform-elevate/elevate-core-ui
```

### Basic Usage

```html
<!-- Status badges -->
<elvt-badge variant="success">Active</elvt-badge>
<elvt-badge variant="warning">Pending</elvt-badge>
<elvt-badge variant="danger">Error</elvt-badge>

<!-- Count badges -->
<elvt-badge variant="info">5</elvt-badge>
<elvt-badge variant="danger">99+</elvt-badge>

<!-- Outline badges -->
<elvt-badge variant="outline">Tag</elvt-badge>
<elvt-badge variant="outline">Category</elvt-badge>
```

### Key Properties

| Property | Type | Description |
|----------|------|-------------|
| `variant` | `success`, `warning`, `danger`, `info`, `neutral`, `outline` | Badge color variant |
| `size` | `small`, `medium` (default), `large` | Visual size of badge |
| `dismissible` | Boolean | Whether badge can be closed/removed |
| `icon` | String | Icon name to display before text |

*For complete API reference including all properties, events, and methods, see the [Storybook documentation](https://elevate-core-ui.inform-cloud.io/?path=/docs/components-badge--docs).*

## Accessibility

### Essential Accessibility Practices

**Color and contrast:**
- Ensure sufficient color contrast for all text (4.5:1 minimum)
- Don't rely solely on color to convey meaning
- Provide alternative indicators like icons or text
- Test badges in high contrast mode

**Screen reader support:**
- Use appropriate ARIA labels for context
- Provide meaningful text for count badges ("3 unread messages")
- Ensure badge purpose is clear from content alone
- Include status information in labels when needed
