# Select

Select components provide efficient option selection through dropdown interfaces that save screen space while offering clear choices. The ELEVATE Select component helps designers and developers create consistent, accessible selection interfaces that work well for both simple and complex option sets.

<TocLinks />

## Overview

### When to Use Select
- **Limited screen space**: When radio buttons would take too much vertical space
- **Many options**: Lists with 7+ options that need to be scannable
- **Familiar patterns**: Standard form fields like country, state, category selection
- **Optional selection**: When users don't need to see all options immediately
- **Searchable content**: Large option lists that benefit from type-ahead filtering

### When NOT to Use Select
- **2-3 options**: Use Radio buttons for few mutually exclusive choices
- **Multiple selection**: Use Checkbox groups for selecting multiple items
- **Boolean choices**: Use Switch component for simple on/off states
- **Primary actions**: Use Button component for actions that trigger processes
- **Always visible options**: When users need to see all choices without interaction

## Design Principles

### Clarity Principle
**Options must be clearly labeled and easy to understand.** Users should be able to quickly scan and identify the option they need without confusion or ambiguity about what each choice means.

### Efficiency Principle
**Selection should be quick and require minimal interaction.** The interface should help users find their desired option quickly, especially in large option lists through search, grouping, or logical ordering.

### Predictability Principle
**Selection behavior should be consistent and expected.** Users should understand how the select will behave based on common patterns and previous experience with similar interfaces.

### Accessibility Principle
**All users must be able to perceive, understand, and operate selections.** This includes keyboard navigation, screen reader support, proper labeling, and sufficient visual contrast.

## Best Practices

### Writing Effective Option Labels

**Why this matters:** Unclear option labels force users to guess or spend time figuring out which option matches their intent, leading to errors and frustration.

:::tip Do: Use clear, descriptive option text
Write option labels that clearly describe what the user is selecting.

```html
<elvt-select name="shipping" placeholder="Choose shipping speed">
  <label slot="label">Shipping Method</label>
  <elvt-option value="standard">Standard (5-7 business days)</elvt-option>
  <elvt-option value="express">Express (2-3 business days)</elvt-option>
  <elvt-option value="overnight">Overnight (next business day)</elvt-option>
</elvt-select>
```
:::

:::danger Don't: Use vague or unclear option labels
Avoid generic labels that don't help users understand their choices.

```html
<!-- Avoid this -->
<elvt-select name="shipping" placeholder="Choose option">
  <label slot="label">Shipping</label>
  <elvt-option value="option1">Option 1</elvt-option>
  <elvt-option value="option2">Option 2</elvt-option>
  <elvt-option value="option3">Option 3</elvt-option>
</elvt-select>
```
:::

### Organizing Large Option Lists

**Why this matters:** Long, unorganized lists become difficult to scan and slow down user task completion. Good organization helps users find options quickly.

:::tip Do: Group related options logically
Organize options in a way that makes sense to users and helps them find choices quickly.

```html
<elvt-select name="location" placeholder="Select location">
  <label slot="label">Office Location</label>
  <optgroup label="North America">
    <elvt-option value="us-ny">New York, USA</elvt-option>
    <elvt-option value="us-ca">San Francisco, USA</elvt-option>
    <elvt-option value="ca-to">Toronto, Canada</elvt-option>
  </optgroup>
  <optgroup label="Europe">
    <elvt-option value="uk-ln">London, UK</elvt-option>
    <elvt-option value="de-be">Berlin, Germany</elvt-option>
    <elvt-option value="fr-pa">Paris, France</elvt-option>
  </optgroup>
</elvt-select>
```
:::

:::danger Don't: Present long lists without organization
Avoid alphabetical-only sorting when logical grouping would be more helpful.

```html
<!-- Avoid this for location-based selections -->
<elvt-select name="location" placeholder="Select location">
  <label slot="label">Office Location</label>
  <elvt-option value="de-be">Berlin, Germany</elvt-option>
  <elvt-option value="uk-ln">London, UK</elvt-option>
  <elvt-option value="us-ny">New York, USA</elvt-option>
  <elvt-option value="fr-pa">Paris, France</elvt-option>
  <elvt-option value="us-ca">San Francisco, USA</elvt-option>
  <elvt-option value="ca-to">Toronto, Canada</elvt-option>
</elvt-select>
```
:::

### Providing Helpful Default States

**Why this matters:** Good defaults reduce cognitive load and help users complete tasks faster. Poor defaults force users to make unnecessary decisions or selections.

:::tip Do: Use meaningful placeholders and defaults
Provide helpful placeholder text and select sensible defaults when appropriate.

```html
<elvt-select name="country" placeholder="Select your country" value="us">
  <label slot="label">Country</label>
  <elvt-option value="us">United States</elvt-option>
  <elvt-option value="ca">Canada</elvt-option>
  <elvt-option value="uk">United Kingdom</elvt-option>
</elvt-select>
```
:::

:::danger Don't: Use generic placeholders
Avoid placeholders that don't help users understand what to select.

```html
<!-- Avoid this -->
<elvt-select name="country" placeholder="Choose">
  <label slot="label">Country</label>
  <elvt-option value="us">United States</elvt-option>
  <elvt-option value="ca">Canada</elvt-option>
  <elvt-option value="uk">United Kingdom</elvt-option>
</elvt-select>
```
:::

### Common Usage Patterns

```html
<!-- Basic select -->
<elvt-select name="category" placeholder="Choose category">
  <label slot="label">Category</label>
  <elvt-option value="tech">Technology</elvt-option>
  <elvt-option value="design">Design</elvt-option>
  <elvt-option value="business">Business</elvt-option>
</elvt-select>

<!-- Select with help text -->
<elvt-select name="priority" placeholder="Select priority" help-text="This affects notification settings">
  <label slot="label">Priority Level</label>
  <elvt-option value="low">Low</elvt-option>
  <elvt-option value="medium">Medium</elvt-option>
  <elvt-option value="high">High</elvt-option>
</elvt-select>

<!-- Required select with validation -->
<elvt-select name="country" required invalid help-text="Please select your country">
  <label slot="label">Country</label>
  <elvt-option value="us">United States</elvt-option>
  <elvt-option value="ca">Canada</elvt-option>
  <elvt-option value="uk">United Kingdom</elvt-option>
</elvt-select>

<!-- Select with grouped options -->
<elvt-select name="timezone" placeholder="Select timezone">
  <label slot="label">Timezone</label>
  <optgroup label="US Timezones">
    <elvt-option value="est">Eastern</elvt-option>
    <elvt-option value="cst">Central</elvt-option>
    <elvt-option value="pst">Pacific</elvt-option>
  </optgroup>
  <optgroup label="International">
    <elvt-option value="gmt">GMT</elvt-option>
    <elvt-option value="cet">CET</elvt-option>
  </optgroup>
</elvt-select>
```

### Key Properties

| Property | Type | Description |
|----------|------|-------------|
| `name` | String | Form field name for submission |
| `value` | String | Currently selected option value |
| `placeholder` | String | Placeholder text when no option is selected |
| `help-text` | String | Additional guidance or validation message |
| `size` | `small`, `medium` (default), `large` | Visual size of the select |
| `required` | Boolean | Whether selection is required |
| `disabled` | Boolean | Whether the select is disabled |
| `invalid` | Boolean | Shows error validation state |
| `multiple` | Boolean | Allow multiple option selection |
| `searchable` | Boolean | Enable type-ahead search in options |

**Slots:** Use `slot="label"` for the select label.

*For complete API reference including all properties, events, and methods, see the [Storybook documentation](https://elevate-core-ui.inform-cloud.io/?path=/docs/components-forms-select--docs).*

## Accessibility

### Essential Accessibility Practices

**Labels are required:**
```html
<!-- Good: Always include labels -->
<elvt-select name="category" placeholder="Choose category">
  <label slot="label">Category</label>
  <elvt-option value="tech">Technology</elvt-option>
</elvt-select>

<!-- Bad: Missing label -->
<elvt-select name="category" placeholder="Choose category">
  <elvt-option value="tech">Technology</elvt-option>
</elvt-select>
```

**Keyboard navigation:**
- **Arrow keys** navigate through options
- **Enter** or **Space** selects the highlighted option
- **Escape** closes the dropdown
- **Tab** moves focus to the next element

**Screen reader support:**
- Labels are announced automatically
- Selected values are communicated clearly
- Help text is associated with the select
- Options are properly announced during navigation

**Visual accessibility:**
- Sufficient color contrast for all text
- Clear focus indicators
- Consistent visual states for different interactions
