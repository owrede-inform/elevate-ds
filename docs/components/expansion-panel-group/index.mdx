# Expansion Panel Group

Expansion Panel Group provides a container for multiple expansion panels with coordinated behavior, consistent styling, and advanced features like accordion mode where only one panel can be expanded at a time.

## When to Use

- **Settings Pages**: Group related configuration sections
- **FAQ Sections**: Organize questions and answers with accordion behavior
- **Multi-Step Forms**: Break complex forms into manageable sections
- **Content Organization**: Structure dense information into collapsible sections

## Properties

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `accordion` | `boolean` | `false` | Enable accordion mode (only one panel expanded) |
| `variant` | `'default' \| 'bordered' \| 'elevated'` | `'default'` | Visual style variant |
| `size` | `'small' \| 'medium' \| 'large'` | `'medium'` | Overall size of panels in the group |
| `gap` | `'none' \| 'small' \| 'medium' \| 'large'` | `'small'` | Spacing between panels |
| `disabled` | `boolean` | `false` | Disables all panels in the group |

## Events

| Event | Detail Type | Description |
|-------|-------------|-------------|
| `elvt-panel-expand` | `CustomEvent<{ panelId: string, expanded: boolean }>` | Fired when any panel expands or collapses |
| `elvt-active-panel-change` | `CustomEvent<{ activePanel: string \| null }>` | Fired when active panel changes (accordion mode) |

## Methods

| Method | Parameters | Description |
|--------|------------|-------------|
| `expandPanel(panelId)` | `panelId: string` | Programmatically expand a specific panel |
| `collapsePanel(panelId)` | `panelId: string` | Programmatically collapse a specific panel |
| `collapseAll()` | - | Collapse all panels in the group |
| `getExpandedPanels()` | - | Returns array of expanded panel IDs |

## Framework Integration

### React Usage

```jsx
import { ElvtExpansionPanelGroup, ElvtExpansionPanel } from '@inform-elevate/elevate-core-ui/dist/react';

function SettingsPage() {
  const [activePanel, setActivePanel] = useState('general');

  const handlePanelChange = (event) => {
    setActivePanel(event.detail.activePanel);
  };

  return (
    <ElvtExpansionPanelGroup 
      accordion 
      onElvtActivePanelChange={handlePanelChange}
    >
      <ElvtExpansionPanel 
        id="general" 
        expanded={activePanel === 'general'}
      >
        <span slot="header">General Settings</span>
        <div slot="content">
          <GeneralSettings />
        </div>
      </ElvtExpansionPanel>
      
      <ElvtExpansionPanel 
        id="privacy" 
        expanded={activePanel === 'privacy'}
      >
        <span slot="header">Privacy Settings</span>
        <div slot="content">
          <PrivacySettings />
        </div>
      </ElvtExpansionPanel>
    </ElvtExpansionPanelGroup>
  );
}
```

### Vue Usage

```vue
<template>
  <elvt-expansion-panel-group 
    :accordion="true"
    @elvt-panel-expand="handlePanelExpand"
  >
    <elvt-expansion-panel
      v-for="section in settingsSections"
      :key="section.id"
      :id="section.id"
      :expanded="expandedPanels.includes(section.id)"
    >
      <span slot="header">{{ section.title }}</span>
      <span slot="description">{{ section.description }}</span>
      <div slot="content">
        <component :is="section.component" />
      </div>
    </elvt-expansion-panel>
  </elvt-expansion-panel-group>
</template>

<script>
export default {
  data() {
    return {
      expandedPanels: ['general'],
      settingsSections: [
        { id: 'general', title: 'General', description: 'Basic settings', component: 'GeneralSettings' },
        { id: 'privacy', title: 'Privacy', description: 'Privacy preferences', component: 'PrivacySettings' }
      ]
    };
  },
  methods: {
    handlePanelExpand(event) {
      const { panelId, expanded } = event.detail;
      if (expanded) {
        this.expandedPanels = [panelId]; // Accordion mode
      } else {
        this.expandedPanels = [];
      }
    }
  }
};
</script>
```

## Accessibility

- **Keyboard Navigation**: Arrow keys navigate between panel headers
- **ARIA Attributes**: Proper `role="tablist"` and `role="tab"` attributes
- **Screen Reader Support**: Announces panel states and content
- **Focus Management**: Focus moves logically through panel content

## Best Practices

### Content Organization

```jsx
// ‚úÖ Good - Related content grouped together
<ElvtExpansionPanelGroup>
  <ElvtExpansionPanel>
    <span slot="header">Account Information</span>
    <div slot="content">
      <EmailSettings />
      <PasswordSettings />
      <ProfileSettings />
    </div>
  </ElvtExpansionPanel>
</ElvtExpansionPanelGroup>

// ‚ùå Avoid - Mixing unrelated content
<ElvtExpansionPanelGroup>
  <ElvtExpansionPanel>
    <span slot="header">Mixed Settings</span>
    <div slot="content">
      <EmailSettings />
      <GameSettings />
      <BillingSettings />
    </div>
  </ElvtExpansionPanel>
</ElvtExpansionPanelGroup>
```

### Accordion Usage

```jsx
// ‚úÖ Good - Use accordion for FAQ sections
<ElvtExpansionPanelGroup accordion>
  {faqs.map(faq => (
    <ElvtExpansionPanel key={faq.id}>
      <span slot="header">{faq.question}</span>
      <div slot="content">{faq.answer}</div>
    </ElvtExpansionPanel>
  ))}
</ElvtExpansionPanelGroup>

// ‚úÖ Good - Multiple panels for settings sections
<ElvtExpansionPanelGroup>
  {settingsSections.map(section => (
    <ElvtExpansionPanel key={section.id}>
      <span slot="header">{section.title}</span>
      <div slot="content">{section.content}</div>
    </ElvtExpansionPanel>
  ))}
</ElvtExpansionPanelGroup>
```

---

:::tip Accordion vs Multiple
Use accordion mode for FAQ sections where users typically need to see only one answer at a time. Use multiple expansion for settings where users might need to compare or work with multiple sections simultaneously.
:::

:::info Performance
For groups with many panels containing heavy content, consider lazy loading panel content only when expanded to improve initial page performance.
:::
