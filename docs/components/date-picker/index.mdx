# Date Picker

Date Picker simplifies date selection for users while ensuring proper formatting and validation. The ELEVATE Date Picker component helps designers and developers create consistent, accessible date input experiences that work across different locales and use cases.

## Overview

### When to Use Date Picker
- **Event scheduling**: Appointments, meetings, and calendar events
- **Booking systems**: Travel, reservations, and service bookings
- **Data filtering**: Date range filters for reports and analytics
- **Form inputs**: Birth dates, deadlines, and important dates
- **Time-sensitive content**: Publication dates and expiration times

### When NOT to Use Date Picker
- **Approximate dates**: Use text input for "around 2020" or vague timeframes
- **Historical dates**: Very old dates may be better as separate year/month inputs
- **Time-only selection**: Use Time Picker component for time without dates
- **Simple text**: When users prefer typing familiar formats like "tomorrow"

## Design Principles

### Clarity Principle
**Date formats should be clearly indicated and consistently applied.** Users should understand the expected format and see examples to prevent input errors and confusion.

### Efficiency Principle
**Date selection should require minimal effort and provide shortcuts.** Quick selection methods for common dates (today, tomorrow, next week) help users complete tasks faster.

### Validation Principle
**Date inputs should provide immediate feedback about validity.** Real-time validation prevents form submission errors and guides users toward correct input.

### Localization Principle
**Date formats should match user locale and cultural expectations.** Proper localization ensures dates display in familiar formats and respect cultural conventions.

## Best Practices

### Providing Clear Date Formats

**Why this matters:** Date formats vary significantly across regions and contexts. Clear format indication prevents user confusion and input errors.

:::tip Do: Show format examples and provide hints
Make the expected date format clear with examples and placeholder text.

```html
<label for="event-date">Event Date</label>
<input 
  type="date" 
  id="event-date"
  placeholder="MM/DD/YYYY"
  aria-describedby="date-format"
/>
<span id="date-format">Format: MM/DD/YYYY (e.g., 12/25/2025)</span>
```
:::

:::danger Don't: Use ambiguous formats without guidance
Avoid date inputs without clear format indication or examples.

```html
<!-- Avoid this -->
<label for="date">Date</label>
<input type="date" id="date" />
```
:::

### Setting Appropriate Constraints

**Why this matters:** Date constraints help users select valid dates and prevent impossible or illogical selections that would cause form errors.

:::tip Do: Set logical min/max dates and provide clear constraints
Use date constraints that make sense for the context and explain limitations.

```html
<label for="birth-date">Birth Date</label>
<input 
  type="date" 
  id="birth-date"
  max="2007-08-10"
  min="1900-01-01"
  aria-describedby="birth-date-help"
/>
<span id="birth-date-help">Must be 18 years or older</span>
```
:::

:::danger Don't: Allow impossible date selections
Avoid date inputs without appropriate constraints for the context.

```html
<!-- Avoid this for age verification -->
<label for="birth-date">Birth Date (must be 18+)</label>
<input type="date" id="birth-date" />
```
:::

## Implementation

### Getting Started

```bash
npm install @inform-elevate/elevate-core-ui
```

### Basic Usage

```html
<!-- Basic date picker -->
<elvt-date-picker 
  name="event-date" 
  label="Event Date"
  placeholder="MM/DD/YYYY">
</elvt-date-picker>

<!-- Date picker with constraints -->
<elvt-date-picker 
  name="birth-date" 
  label="Birth Date"
  max="2007-08-10"
  help-text="Must be 18 years or older">
</elvt-date-picker>

<!-- Required date picker -->
<elvt-date-picker 
  name="deadline" 
  label="Project Deadline"
  required
  min="2025-08-10">
</elvt-date-picker>

<!-- Date range picker -->
<elvt-date-range-picker 
  start-name="check-in"
  end-name="check-out"
  start-label="Check-in Date"
  end-label="Check-out Date"
  min="2025-08-10">
</elvt-date-range-picker>
```

### Key Properties

| Property | Type | Description |
|----------|------|-------------|
| `name` | String | Form field name for data submission |
| `value` | String | Currently selected date (YYYY-MM-DD format) |
| `label` | String | Label text for the date picker |
| `placeholder` | String | Placeholder text showing expected format |
| `min` | String | Minimum selectable date (YYYY-MM-DD format) |
| `max` | String | Maximum selectable date (YYYY-MM-DD format) |
| `required` | Boolean | Whether date selection is required |
| `disabled` | Boolean | Whether the date picker is disabled |
| `help-text` | String | Additional guidance or validation message |
| `error-message` | String | Error message for validation failures |

*For complete API reference including all properties, events, and methods, see the [Storybook documentation](https://elevate-core-ui.inform-cloud.io/?path=/docs/components-date-picker--docs).*

## Accessibility

### Essential Accessibility Practices

**Keyboard navigation:**
- Tab moves focus between date picker and other form elements
- Arrow keys navigate calendar dates when calendar is open
- Enter selects the focused date
- Escape closes calendar popup without selection

**Screen reader support:**
- Date picker labels are properly announced
- Selected dates are communicated clearly
- Calendar navigation is announced (month, year changes)
- Date constraints and validation messages are announced

**Visual accessibility:**
- Sufficient color contrast for all text and interactive elements
- Clear focus indicators for keyboard navigation
- Selected dates are visually distinct
- Error states use color plus text/icons (not color alone)

**Input flexibility:**
- Support both keyboard input and mouse/touch interaction
- Allow manual typing for users who prefer text entry
- Provide format examples and validation feedback
- Support common date formats and abbreviations
