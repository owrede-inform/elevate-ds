# Card

Cards are versatile containers that organize related information and actions into digestible sections. The ELEVATE Card component helps designers and developers create consistent, accessible layouts that structure complex information clearly and support user workflows.

<TocLinks />

## Overview

### When to Use Cards
- **Content grouping**: Related information that belongs together logically
- **Data display**: Metrics, statistics, dashboards, or summary information
- **Navigation**: Category links, feature highlights, landing page sections
- **Product listings**: Items in catalogs, directories, or search results
- **Information hierarchy**: Breaking complex content into digestible sections

### When NOT to Use Cards
- **Simple lists**: Use List component for basic item collections
- **Form sections**: Use proper form fieldsets or sections for form organization
- **Full-page layouts**: Use appropriate layout components for main page structure
- **Inline content**: Use appropriate text components for paragraph-level information

## Design Principles

Understanding these core principles will help you create card layouts that enhance rather than complicate information consumption and user task completion.

### Information Hierarchy Principle
**Cards should create clear visual hierarchy within content.** Users need to quickly scan and prioritize information within cards. Poor hierarchy forces users to read everything to find what they need, slowing down task completion.

### Content Cohesion Principle  
**Related information should be grouped together logically.** Cards work best when they contain information that naturally belongs together. Mixing unrelated content confuses users and breaks their mental models of how information is organized.

### Visual Consistency Principle
**Similar content types should use similar card treatments.** Consistent card patterns help users predict where to find information and how to interact with content. Visual inconsistency creates cognitive overhead and reduces interface familiarity.

### Contextual Appropriateness Principle
**Card treatment should match the content's importance and purpose.** High-priority content may need elevated cards, while background information works better with subtle treatments. Mismatched visual weight creates confusion about content importance.

## Best Practices

### Creating Effective Content Hierarchy

**Why this matters:** Cards without clear visual hierarchy force users to read all content to find relevant information. Good hierarchy enables quick scanning and helps users prioritize their attention effectively.

:::tip Do: Use clear headings and visual hierarchy
Structure card content with proper headings, logical spacing, and visual emphasis.

```html
<elvt-card>
  <h3>Project Alpha</h3>
  <p class="meta">Status: In Progress • Due: March 15</p>
  <p>Backend API development is 80% complete. Frontend integration begins next week.</p>
  <elvt-button tone="primary">View Details</elvt-button>
</elvt-card>
```
:::

:::danger Don't: Use flat hierarchy
Avoid flat text structure that makes content difficult to scan and prioritize.

```html
<!-- Avoid this -->
<elvt-card>
  <p>Project Alpha</p>
  <p>Status: In Progress</p>
  <p>Due: March 15</p>
  <p>Backend API development is 80% complete...</p>
  <elvt-button>View Details</elvt-button>
</elvt-card>
```
:::

### Grouping Related Content Effectively

**Why this matters:** Cards that mix unrelated information increase cognitive load and make it harder for users to understand the purpose of each card. Logical grouping helps users process information more efficiently.

:::tip Do: Group logically related information
Keep related information together that serves a common purpose.

```html
<elvt-card>
  <h3>Account Security</h3>
  <div class="security-items">
    <div class="item">Two-factor Authentication: Enabled</div>
    <div class="item">Password Strength: Strong</div>
    <div class="item">Last Login: 2 hours ago</div>
  </div>
  <elvt-button tone="primary">Security Settings</elvt-button>
</elvt-card>
```
:::

:::danger Don't: Mix unrelated information
Avoid combining unrelated content that serves different purposes.

```html
<!-- Avoid this -->
<elvt-card>
  <h3>Mixed Information</h3>
  <p>Account balance: $1,234.56</p>
  <p>Weather today: Sunny, 72°F</p>
  <p>Unread messages: 5</p>
  <p>System update available</p>
</elvt-card>
```
:::

```bash
npm install @inform-elevate/elevate-core-ui
```

```javascript
// Import all components

```

### Common Usage Patterns

```html
<!-- Basic content card -->
<elvt-card>
  <h3>Card Title</h3>
  <p>Card content goes here.</p>
  <elvt-button tone="primary">Action</elvt-button>
</elvt-card>

<!-- Card with header and footer -->
<elvt-card>
  <div slot="header">
    <h3>Header Content</h3>
  </div>
  
  <p>Main content area</p>
  
  <div slot="footer">
    <elvt-button tone="primary">Primary Action</elvt-button>
    <elvt-button>Secondary Action</elvt-button>
  </div>
</elvt-card>

<!-- Elevated card with custom padding -->
<elvt-card layer="elevated" padding="l">
  <h3>Important Information</h3>
  <p>This card stands out with elevation and generous padding.</p>
</elvt-card>

<!-- Card grid layout -->
<div class="card-grid">
  <elvt-card>Card 1</elvt-card>
  <elvt-card>Card 2</elvt-card>
  <elvt-card>Card 3</elvt-card>
</div>
```

### Key Properties

| Property | Options | Description |
|----------|---------|-------------|
| `padding` | `s`, `m` (default), `l`, `xl` | Internal spacing amount |
| `border` | `all` (default), `rounded`, `none` | Border treatment style |
| `layer` | `default`, `elevated`, `overlay`, `sunken` | Visual depth treatment |
| `background` | `default`, `subtle`, `accent` | Background color variant |

**Slots:** Use `slot="header"` and `slot="footer"` for structured layout sections.

*For complete API reference including all properties, events, and methods, see the [Storybook documentation](https://elevate-core-ui.inform-cloud.io/?path=/docs/components-layout-card--docs).*

## Accessibility

### Essential Accessibility Practices

**Semantic structure:**
```html
<!-- Good: Proper heading hierarchy -->
<elvt-card>
  <h3>Main Heading</h3>
  <h4>Subheading</h4>
  <p>Content text</p>
</elvt-card>
```

**Focus management:**
- Cards with interactive elements should allow proper tab navigation
- Ensure sufficient color contrast for all text content
- Use appropriate ARIA labels when cards represent complex structures

**Keyboard interaction:**
- All interactive elements within cards should be keyboard accessible
- Consider card-level keyboard shortcuts for complex interactions
