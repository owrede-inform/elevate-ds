# ELEVATE Components

The ELEVATE Component Library provides a comprehensive set of accessible, consistent, and well-documented UI components for building modern web applications. Each component follows ELEVATE design principles and integrates seamlessly with the INFORM design system.

## Component Categories

- **Menu** - Application navigation menus

## Getting Started

### Installation

```bash
npm install @inform-elevate/elevate-core-ui
```

### Basic Usage

```javascript
// Import all components
import '@inform-elevate/elevate-core-ui';

// Or import specific components
import '@inform-elevate/elevate-core-ui/dist/components/button/button.js';
import '@inform-elevate/elevate-core-ui/dist/components/input/input.js';
```

### HTML Usage

```html
<!-- Use ELEVATE components as web components -->
<elvt-button tone="primary">Click me</elvt-button>

<elvt-input 
  type="email" 
  label="Email Address" 
  placeholder="user@example.com"
  required>
</elvt-input>

<elvt-card padding="l">
  <h3>Card Title</h3>
  <p>Card content goes here.</p>
  <elvt-button tone="primary">Action</elvt-button>
</elvt-card>
```

### React Integration

```jsx
import { ElvtButton, ElvtInput, ElvtCard } from '@inform-elevate/elevate-core-ui/dist/react';

function MyComponent() {
  return (
    <>
      <ElvtButton tone="primary" onClick={handleClick}>
        Click me
      </ElvtButton>
      
      <ElvtInput 
        type="email"
        label="Email Address"
        placeholder="user@example.com"
        required
      />
      
      <ElvtCard padding="l">
        <h3>Card Title</h3>
        <p>Card content goes here.</p>
      </ElvtCard>
    </>
  );
}
```

## Design System Integration

ELEVATE components are built with design system principles at their core:

### Design Tokens
All components use ELEVATE design tokens for consistent:
- **Colors** - Semantic color palette with light/dark themes
- **Typography** - Type scales and font families  
- **Spacing** - Consistent spacing units and padding options
- **Borders** - Border radius, widths, and styles
- **Shadows** - Elevation and depth treatments

### Accessibility Standards
Every component meets WCAG 2.1 AA standards:
- **Keyboard Navigation** - Full keyboard accessibility
- **Screen Reader Support** - Proper ARIA attributes and labels
- **Color Contrast** - Sufficient contrast ratios for all text
- **Focus Management** - Clear focus indicators and logical tab order

### Responsive Design
Components adapt to different screen sizes:
- **Mobile-First** - Optimized touch targets and layouts
- **Flexible Layouts** - Components that work in various containers
- **Breakpoint Awareness** - Responsive behavior when appropriate

## Documentation Standards

Each component includes comprehensive documentation:
- **Usage Guidelines** - When and how to use the component
- **API Reference** - Complete property, event, and method documentation  
- **Design Principles** - Underlying design philosophy and rationale
- **Best Practices** - Recommended usage patterns with examples
- **Accessibility** - Specific accessibility features and requirements
- **Interactive Examples** - Live component demonstrations
