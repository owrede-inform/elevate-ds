# Empty State

Empty State components provide helpful placeholders when no content, data, or search results are available. They guide users with clear messaging and actionable next steps to improve the user experience during empty scenarios.

<TocLinks />

## When to Use

- **No Data**: When lists, tables, or data views have no items
- **Search Results**: When search queries return no matches
- **First Time Use**: When users haven't created content yet
- **Error Recovery**: After failed operations that result in empty states

## Properties

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `icon` | `string` | | Icon name from ELEVATE icon library |
| `title` | `string` | | Main heading text |
| `description` | `string` | | Descriptive text explaining the empty state |
| `action-text` | `string` | | Text for the primary action button |
| `action-href` | `string` | | URL for the action button (makes it a link) |
| `variant` | `'default' \| 'search' \| 'error' \| 'welcome'` | `'default'` | Visual style variant |
| `size` | `'small' \| 'medium' \| 'large'` | `'medium'` | Overall size of the empty state |

## Events

| Event | Detail Type | Description |
|-------|-------------|-------------|
| `elvt-action-click` | `CustomEvent<void>` | Fired when action button is clicked |

## Variants

### Default
Standard empty state for general no-data scenarios.

### Search
Optimized for search result pages with search-specific messaging.

### Error
For error states that result in empty content with error recovery actions.

### Welcome
Positive, welcoming tone for first-time user experiences.

## Framework Integration

### React Usage

```jsx
import { ElvtEmptyState } from '@inform-elevate/elevate-core-ui/dist/react';

function DataTable({ data }) {
  if (data.length === 0) {
    return (
      <ElvtEmptyState
        icon="table"
        title="No data available"
        description="Import data or create new entries to get started."
        actionText="Import data"
        onElvtActionClick={handleImport}
      />
    );
  }

  return <Table data={data} />;
}
```

### HTML Usage

```html
<elvt-empty-state
  icon="inbox"
  title="No notifications"
  description="You're all caught up! Check back later for updates."
  action-text="Go to settings"
  action-href="/settings">
</elvt-empty-state>
```

### Vue Usage

```vue
<template>
  <elvt-empty-state
    v-if="items.length === 0"
    icon="list"
    title="No items found"
    description="Create your first item to get started."
    action-text="Add item"
    @elvt-action-click="addItem"
  />
</template>
```

## Best Practices

### Content Guidelines

```jsx
// ✅ Good - Helpful and actionable
<ElvtEmptyState
  title="No projects yet"
  description="Create your first project to start tracking tasks and milestones."
  actionText="Create project"
/>

// ❌ Avoid - Vague or unhelpful
<ElvtEmptyState
  title="Empty"
  description="Nothing here."
  actionText="Do something"
/>
```

### Context-Specific Messaging

```jsx
// ✅ Good - Specific to the context
<ElvtEmptyState
  icon="search"
  title="No results for 'design system'"
  description="Try different keywords or check your spelling."
  actionText="Clear search"
  variant="search"
/>

// ✅ Good - First-time user guidance
<ElvtEmptyState
  icon="star"
  title="Welcome to your workspace!"
  description="Invite team members and create your first project."
  actionText="Get started"
  variant="welcome"
/>
```

### Action Prioritization

```jsx
// ✅ Good - Clear primary action
<ElvtEmptyState
  title="No team members"
  description="Collaborate by inviting team members to your workspace."
  actionText="Invite members"
  actionHref="/team/invite"
/>
```
