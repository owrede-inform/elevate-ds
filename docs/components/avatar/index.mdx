# Avatar

Avatars are visual representations of users, teams, or entities within interfaces. The ELEVATE Avatar component helps designers and developers create consistent, accessible identity systems that enhance user recognition and provide clear visual identification across all interface contexts.

## Overview

### When to Use Avatars
- **User identification**: Profile areas, user lists, comments sections, and author attribution
- **Team representation**: Member lists, collaboration interfaces, and group displays
- **Entity representation**: Organizations, projects, departments, or system accounts
- **Navigation elements**: User menus, account switching, and profile access points
- **Social features**: Comments, reviews, messaging, and activity feeds
- **Contact lists**: Address books, directory listings, and people browsing

### When NOT to Use Avatars
- **Decorative images**: Use Image component for non-representative visuals or illustrations
- **Large portraits**: Use dedicated image layouts for hero content or detailed profile photos
- **Complex graphics**: Use appropriate media components for logos, diagrams, or artwork
- **Brand representation**: Use dedicated logo or brand components for company identification
- **Non-human entities**: Consider appropriate icon or image components for objects or concepts

## Design Principles

### Recognition Principle
**Avatars should facilitate quick user identification and recognition.** Consistent representation helps users identify specific individuals or entities across different interface contexts.

### Clarity Principle
**Avatar content should be legible and appropriate for its size context.** Text, initials, and icons must remain readable at all supported sizes while maintaining visual quality.

### Accessibility Principle
**All users must be able to perceive and understand avatar content.** This includes proper alternative text, sufficient contrast, and support for assistive technologies.

### Scalability Principle
**Avatars must work effectively across different sizes and contexts.** The same avatar should provide appropriate detail and readability whether used in lists, headers, or detailed views.

## Best Practices

### Choosing Appropriate Fallbacks

**Why this matters:** Not all users will have profile images. Clear fallback strategies ensure consistent user identification even without uploaded photos.

:::tip Do: Use meaningful fallbacks in order of preference
Implement a clear hierarchy: image ‚Üí initials ‚Üí icon, with appropriate styling for each.

```html
<!-- Good fallback hierarchy -->
<elvt-avatar 
  src="user-photo.jpg"
  alt="John Doe"
  initials="JD"
  fallback-icon="user">
</elvt-avatar>
```
:::

:::danger Don't: Use generic placeholders without context
Avoid meaningless placeholder images or generic icons that don't help identify users.

```html
<!-- Avoid this -->
<elvt-avatar src="generic-user.png" alt="User">
</elvt-avatar>
```
:::

### Ensuring Proper Contrast

**Why this matters:** Avatar content must be readable against background colors. Poor contrast makes avatars ineffective for identification purposes.

:::tip Do: Test contrast ratios for all avatar states
Ensure text and icons meet WCAG contrast requirements against background colors.

```html
<elvt-avatar 
  initials="AB" 
  background-color="#1f2937"
  text-color="#ffffff">
</elvt-avatar>
```
:::

:::danger Don't: Use low contrast color combinations
Avoid color combinations that make text or icons difficult to read.

```html
<!-- Avoid this -->
<elvt-avatar 
  initials="AB" 
  background-color="#f0f0f0"
  text-color="#d0d0d0">
</elvt-avatar>
```
:::

## Implementation

### Getting Started

```bash
npm install @inform-elevate/elevate-core-ui
```

### Basic Usage

```html
<!-- Image avatar -->
<elvt-avatar 
  src="profile-photo.jpg"
  alt="John Doe"
  size="medium">
</elvt-avatar>

<!-- Initials avatar -->
<elvt-avatar 
  initials="JD"
  background-color="#3b82f6"
  size="medium">
</elvt-avatar>

<!-- Icon avatar -->
<elvt-avatar 
  icon="user"
  background-color="#6b7280"
  size="medium">
</elvt-avatar>

<!-- Avatar with fallback -->
<elvt-avatar 
  src="broken-image.jpg"
  alt="Jane Smith"
  initials="JS"
  fallback-icon="user"
  size="large">
</elvt-avatar>
```

### Key Properties

| Property | Type | Description |
|----------|------|-------------|
| `src` | String | Image source URL for photo avatars |
| `alt` | String | Alternative text for accessibility |
| `initials` | String | User initials for text-based avatars |
| `icon` | String | Icon name for icon-based avatars |
| `size` | `xsmall`, `small`, `medium` (default), `large`, `xlarge` | Visual size of avatar |
| `background-color` | String | Background color for non-image avatars |
| `text-color` | String | Text color for initials |
| `shape` | `circle` (default), `square`, `rounded` | Avatar shape variant |

*For complete API reference including all properties, events, and methods, see the [Storybook documentation](https://elevate-core-ui.inform-cloud.io/?path=/docs/components-avatar--docs).*

## Accessibility

### Essential Accessibility Practices

**Alternative text requirements:**
- All avatars must have descriptive alt text
- Image avatars should include the person's name
- Icon avatars should describe their purpose
- Initials avatars should include the full name

**Visual accessibility:**
- Sufficient contrast between text/icons and background (4.5:1 minimum)
- Clear visual boundaries for avatar shapes
- Consistent sizing that maintains readability
- Support for high contrast mode

**Screen reader support:**
- Proper role attribution for avatar elements
- Meaningful descriptions for initials and icon avatars
- Status information when relevant (online, offline, etc.)
- Group context when avatars appear in lists
