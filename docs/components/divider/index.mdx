Dividers provide subtle visual separation that helps users understand content relationships and hierarchy. The ELEVATE Divider component helps designers and developers create consistent, accessible interfaces that organize information clearly without creating visual clutter.

<TocLinks />

## Overview

### When to Use Dividers
- **Content sections**: Separate distinct topics or categories within a page
- **List items**: Create visual breaks between related but separate list entries
- **Form groups**: Organize form fields into logical sections
- **Navigation menus**: Separate menu groups or categories
- **Card content**: Divide different types of information within cards

### When NOT to Use Dividers
- **White space is sufficient**: Don't add dividers when spacing already creates clear separation
- **Overuse**: Too many dividers create visual noise and reduce their effectiveness
- **Primary navigation**: Use proper navigation components instead of divider-separated links
- **Data tables**: Use proper table structure instead of dividers between data rows

### Visual Styles

**Solid Line** provides clear, definitive separation between content sections.

**Dashed Line** creates softer separation while maintaining visual distinction.

**Dotted Line** offers subtle separation for closely related content groups.

**Gradient Fade** provides gentle visual transition between content areas.

### Content Integration

**Plain Dividers** offer simple line separation without additional content.

**Text Labels** include descriptive text within the divider for section identification.

**Icon Integration** combines icons with dividers for enhanced visual communication.

## Design Principles

### Subtlety Principle
**Dividers should organize without overwhelming.** They provide structure through subtle visual cues rather than prominent visual elements. Heavy or overly decorative dividers compete with content for user attention.

### Semantic Purpose Principle  
**Every divider should have a clear organizational reason.** Random or decorative dividers confuse information hierarchy. Each divider should represent a meaningful content boundary that helps users understand relationships.

### Consistency Principle
**Similar content separations should use similar divider treatments.** Inconsistent divider usage creates visual chaos and reduces the user's ability to predict interface patterns.

### Proportional Hierarchy Principle
**Divider visual weight should match the importance of the separation.** Major section breaks might use stronger dividers while minor separations use lighter treatments.

## Best Practices

### Using Appropriate Visual Weight

**Why this matters:** Dividers that are too prominent draw attention away from content, while dividers that are too subtle fail to provide clear organization. The visual weight should match the level of separation needed.

### Spacing and Alignment

**Why this matters:** Inconsistent spacing around dividers creates visual imbalance and reduces their effectiveness as organizational tools. Proper spacing ensures dividers enhance rather than interrupt content flow.

### Avoiding Overuse

**Why this matters:** Too many dividers create visual noise and reduce their effectiveness. When everything is separated, nothing feels organized. Strategic use maintains their organizational value.

### Common Usage Patterns

```html
<!-- Basic horizontal divider -->
<elvt-divider></elvt-divider>

<!-- Vertical divider for inline content -->
<elvt-divider orientation="vertical"></elvt-divider>

<!-- Divider with custom styling -->
<elvt-divider style="height: 2px; background-color: #0f62fe;"></elvt-divider>

<!-- In menu contexts (used automatically in dropdowns) -->
<elvt-dropdown>
  <elvt-button slot="trigger" caret>Menu</elvt-button>
  <elvt-menu-item>Option 1</elvt-menu-item>
  <elvt-menu-item>Option 2</elvt-menu-item>
  <elvt-divider></elvt-divider>
  <elvt-menu-item>Option 3</elvt-menu-item>
</elvt-dropdown>

<!-- In card layouts -->
<div class="card">
  <div class="card-header">Header Content</div>
  <elvt-divider></elvt-divider>
  <div class="card-body">Body Content</div>
</div>
```

### Key Properties

| Property | Options | Description |
|----------|---------|-------------|
| `orientation` | `horizontal` (default), `vertical` | Direction of the divider line |
| `style` | Standard CSS style properties | Apply custom styling to the divider element |

**Usage Notes:**
- The divider inherits styling from the current theme
- Custom colors and thickness can be applied via CSS `style` attribute
- Vertical dividers typically need explicit height styling
- Spacing is managed by parent container layout

*For complete API reference including all properties, events, and methods, see the [Storybook documentation](https://elevate-core-ui.inform-cloud.io/?path=/docs/components-divider--docs).*

## Accessibility

### Essential Accessibility Practices

**Semantic markup considerations:**
- Dividers are primarily visual elements
- Use appropriate ARIA roles when dividers have semantic meaning
- Don't rely on dividers alone to convey important information structure

**Screen reader behavior:**
- Most dividers should be ignored by screen readers
- Use `role="separator"` for dividers that have semantic meaning
- Provide alternative structure cues through proper heading hierarchy

**Focus management:**
- Dividers typically should not receive keyboard focus
- Interactive dividers (like collapsible sections) need proper focus handling
- Ensure content before and after dividers has logical tab order

**Visual accessibility:**
- Ensure sufficient contrast for divider lines (3:1 minimum)
- Don't rely solely on dividers for content organization
- Provide additional visual or structural cues for important boundaries
