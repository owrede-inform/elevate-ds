## Design Token Testing

This page helps verify that ELEVATE design tokens are properly applied:

### âœ… What to Check:
- **Button Tones**: `tone` prop with values: neutral, primary, success, warning, danger, emphasized, subtle
- **Button Properties**: `size`, `disabled`, `selected`, `href` for link buttons
- **Input Types**: text, email, password, search, number, date, tel, url
- **Input Properties**: `label`, `helpText`, `placeholder`, `required`, `disabled`, `readonly`, `clearable`
- **Card Features**: Basic container with optional `padding`, `border`, `layer` properties
- **Form Integration**: Proper form field behavior and validation
- **Design Tokens**: ELEVATE colors, typography, spacing applied consistently
- **Dark Mode**: All components adapt to theme changes

### ðŸ§ª Testing Steps:
1. **Button Interactions**: Test all tone variants, sizes, and states
2. **Input Validation**: Type in fields, test required validation, try different input types
3. **Form Behavior**: Test form submission and field interactions
4. **Theme Toggle**: Use the theme switcher to test light/dark mode adaptation
5. **Responsive Design**: Resize browser to test mobile/desktop layouts
6. **Accessibility**: Navigate using Tab key, test screen reader compatibility

### ðŸ“š API Reference:
- **ElvtButton**: `tone`, `size`, `disabled`, `selected`, `href`, `target`, `download`
- **ElvtInput**: `type`, `label`, `helpText`, `placeholder`, `value`, `required`, `disabled`, `readonly`, `clearable`
- **ElvtCard**: `padding`, `border`, `layer` with slots for content, header, and footer
- **ElvtStack**: `direction`, `gap`, `alignment`, `distribution`, `wrap`, `padding` for flexible layouts
